# 1C Platform CLI Reference

Этот документ дополняет `onec-context-toolkit` знаниями о platform CLI и headless workflow 1С.

Использовать его, когда вопрос касается:

- `1cv8` в режимах `CONFIG`, `ENTERPRISE`, `CREATEINFOBASE`
- пакетных операций конфигуратора и headless/CI-сценариев
- `ragent`, `ras`, `rac`
- `ibsrv`, `ibcmd`, SSH-шлюза автономного сервера
- временных ИБ, `ibases.v8i`, XML import/export, сборки `.epf` / `.cfe`

## Главное правило

CLI 1С сильно зависит от версии платформы. Для точного синтаксиса и полного набора флагов всегда сверять ответ с официальной документацией той же версии, что установлена у пользователя.

## Быстрый выбор инструмента

| Задача | Основной инструмент | Комментарий |
|---|---|---|
| Создать ИБ без GUI | `1cv8 CREATEINFOBASE` | Канонический путь для новой файловой или серверной ИБ. |
| Выполнить пакетную команду конфигуратора | `1cv8 CONFIG` | Нужен контекст ИБ, обычно через `/F <каталог>` или серверную строку подключения. |
| Управлять кластером серверов 1С | `ragent` + `ras` + `rac` | `ragent` — процесс кластера; `ras`/`rac` — административный контур. |
| Управлять автономным сервером и ИБ | `ibsrv` + `ibcmd` | `ibsrv` даёт standalone server; `ibcmd` работает локально, по PID, по SSH или офлайн. |
| Headless XML import/export, сборка `.cfe`, часть операций с ИБ | `ibcmd config ...` | Чаще проще для CI, чем поднимать GUI-конфигуратор. |
| Сборка `.epf` / `.erf` из выгрузки конфигуратора | `1cv8 CONFIG` или `ibcmd config import --out=...` | Выбор зависит от версии платформы и типа входных файлов. |

## Термины, которые нельзя смешивать

- `ragent` — агент сервера кластера. Это не то же самое, что SSH или standalone server.
- `ras` / `rac` — CLI-администрирование кластера через `ras`.
- `ibsrv` — автономный сервер. Это не полноценная замена кластера в production.
- `ibcmd` — утилита администрирования standalone server и ИБ.
- SSH-шлюз `ibsrv` — не обычный shell-доступ и не доступ к PostgreSQL.
- "Конфигуратор в агентском режиме" через SSH относится к `ibsrv`, а не к `ragent`.

## Практические правила

### 1. Для headless и CI почти всегда нужен контекст ИБ

Если выполняется пакетный `CONFIG` или операция, где нужны типы конфигурации, удобнее работать через временную файловую ИБ:

1. Создать её через `1cv8 CREATEINFOBASE`.
2. Не добавлять в `ibases.v8i`, если это чисто автоматический pipeline.
3. Запускать `1cv8 CONFIG /F <temp-ib>` с нужной пакетной командой.
4. После сборки удалить ИБ или сохранить осознанно для повторного использования.

Это особенно важно для сценариев вроде `LoadExternalDataProcessorOrReportFromFiles`, где без ИБ возможны пустой лог или потеря типов.

### 2. Для параллельных задач не делить одну ИБ

Если нужно ускорение в CI:

- не запускать много тяжёлых задач в одну ИБ;
- использовать несколько независимых каталогов ИБ;
- при необходимости запускать несколько `ibsrv` с разными `--data` и портами.

Один `ibsrv` и одна ИБ всё равно упираются в блокировки.

### 3. `ibcmd --remote` использует протокол 1С, а не shell

Внешний `ssh user@host` к SSH-порту `ibsrv` не даёт обычную shell-сессию.
Для удалённого управления standalone server использовать именно `ibcmd --remote ssh://host:port`.

### 4. `rac` и `ras` должны совпадать по линейке версий

При вопросах о кластере всегда проверять совместимость `rac`, `ras` и версии платформы.

### 5. Для скриптов полезны `/Out` и `/DumpResult`

При классическом `1cv8 CONFIG` или `CREATEINFOBASE`:

- писать служебный вывод в файл через `/Out`;
- писать код результата через `/DumpResult`.

Это сильно упрощает разбор ошибок в CI.

## Важные сценарии

### Создание временной ИБ

Использовать `1cv8 CREATEINFOBASE`, когда нужно:

- подготовить лёгкую файловую ИБ для пакетных операций;
- не зависеть от уже существующей боевой базы;
- собирать `.epf` / `.erf` / `.cfe` или загружать XML в headless pipeline.

### Управление standalone server

Использовать `ibsrv` и `ibcmd`, когда нужно:

- поднять автономный сервер;
- подключаться к нему локально по PID;
- подключаться удалённо через SSH-шлюз;
- работать с ИБ офлайн по каталогу данных, если этот каталог не занят другим `ibsrv`.

### Администрирование кластера

Использовать `ragent`, `ras`, `rac`, когда вопрос касается:

- кластеров серверов 1С;
- рабочих процессов и портов;
- сеансов, блокировок, списка ИБ;
- сетевых конфликтов между несколькими кластерами на одной машине.

### Сборка `.epf` / `.erf` / `.cfe`

Практически полезная эвристика такая:

- `1cv8 CONFIG` — хороший путь для классического пакетного конфигуратора;
- `ibcmd config import/export` — хороший headless путь для операций с выгрузкой и ИБ;
- для `.epf` / `.erf` из XML всё равно часто нужен осознанный build path с временной ИБ;
- для Linux GUI-зависимые шаги могут требовать `Xvfb + DISPLAY`.

## Что отвечать аккуратно

Если пользователь спрашивает "какая команда точно нужна", в ответе лучше:

1. определить класс задачи: cluster / standalone / create infobase / package config / XML import-export;
2. назвать правильный инструмент;
3. дать минимальный паттерн запуска;
4. явно сказать, что точные ключи нужно сверить с official docs по той же версии платформы.
